Study S32K3 from Scratch - Software Triggered ADC in Scan Mode (MCAL Version)

Posted on 2022-01-22

Who is this article for?
For people who wants to start S32K3 development.
What is this article about?
In this article, we may not spend too much time on the details of S32K3. Instead, we will configure and debug different applications which utilize different peripherals, learning by doing it is the main theme of my articles.
Why do I write this article?
I struggled a lot at the beginning of learning embedded development, so I hope my articles can help people like me as a green hand, and they are also records of my self-teaching.

Build Developing Environment

If we want to develop our applications on S32K3xx with MCAL, we need S32K3 Real-Time Drivers for Cortex-M, S32 Design Studio 3.4, EB tresos Studio 27.1.0, we can download them in the link below:
S32K3 Microcontrollers for General Purpose
If you encounter problems, please contact NXP’s Account Managers or Technical Sale Representatives for help.

S32K3 Real-Time Drivers for Cortex-M

We can find S32K3 Real-Time Drivers for Cortex-M in the link above, it’s recommended to install it in the default path:

S32 Design Studio 3.4

If we choose MCAL architecture for developing, S32 Design Studio is only for compiling and debugging, thus we don’t the internal development package for S32K3, but we still recommend to intall the latest S32DS S32K3 development package, in case you need to use RTD-LLD for developing one day. There is one thing that we should pay attention, we need GCC 10.2 for compiling:

If your GCC version is not 10.2, you install it in S32 Design Studio’s Help -> S32DS Extensions and Updates:

EB Tresos 27.1

We can download EB Tresos 27.1 and get the license in the link provided above.

Choose the template project, and import the Tresos project inside it:

Double click “someId(CORTEXM, S32K3XX)” to load the module list:

Click “Generate code for the currently selected projects” (little hammer) to generate code:

Here we finish the code generating in EB Tresos 27.1.

Project Compiling in S32 Design Studio 3.4

Click File -> Import -> General -> Existing Projects into Workspace to import the template project, is the RTD is installed in the default path, we don’t need to anything, if the RTD is installed in other paths, we need to change some settings:

  1. Change the link path of MCAL plugins - right-click the template project, choose Properties -> Resource -> Linked Resources -> Linked Resources -> Absolute Path Location -> Link_Source, and change path of MCAL plugins to the actual path;
  2. Change MCAL plugins’ compiling path - right-click the template project, choose Properties -> C/C++ Build -> Build Variables -> MCAL_PLUGIN_PATH, replace it with the actual MCAL plugins path. Click the “Build” button to compile the template project, if everything goes well, “Build Finished. 0 errors, 0 warnings.” will be printed in the Console window, which indicates that the developing environment is successfully built.

Software Triggered Scan Mode ADC Conversion

Software triggered scan mode ADC conversion is relatively simple, we can start to get familiar with S32K3 RTD MCAL by developing it.

EB Tresos Project’s Configuration

Add ADC Module

Right-click the template project and choose “Module Configurations”:

Double-click to add ADC module:

Dio Module’s Configuration

According to the S32K344 MaxQFP172 EVB’s schematic, the input pin of potentiometer is PTA11, ADC1’s S10:

First of all we should configure this ADC channel in Dio module, click Dio -> DioConfig -> DioPort, and we will find A_H,A_L,B_H,B_L…What do they mean? After checking the attachment “S32K344_IOMUX” of S32K3xx’s Reference Manual, we can know that the IO ports are divided into 7 groups according to the A, B, C, D, E, F, G numbers(Devices in MaxQFP package don’t have F and G groups), normally, every group has 32 IOs(from 0 to 31), but some IOs may be missing in smaller packages(For example, MaxQFP172). H and L stand for the upper 16 bits and the lower 16 bits, for example, PTA29, a red LED has already been defined in A_H:

Obviously, it belongs to upper 16 bits, so we defined it in A_H, it’s Dio Channel ID i 29-16=13.
Thus, PTA11 should be defined in A_L, its Dio Channel ID is 11, click Dio -> DioConfig -> DioPort -> A_L -> DioChannel to add a Dio channel and name it PTA11_ADC_POT0, its Channel ID is 11:

Here we’ve finished the Dio module’s configuration according to the actual input.

Port Module’s Configuration

Click Port -> PortConfigSet -> PortContainer -> PortPin, we will find that an IO has already been defined here, we can add a PTA11_ADC1_S10 here:

At this time, we notice that an error was occurred in “PortNumberOfPortPins”, its value is 1, but we’ve add another pin, so we should change it to 2:

After that, we can click PTA11_ADC1_S10 to configure this port pin, there is 2 parameters that we need to modify:

One of the parameters is PortPinMscr, we can find PTA11’s MSCR code is 11 in the IOMUX; another one is PortPin Mode, we should change it to ADC1_ADC1_S10. Now, we’ve finished the Port module’s configuration.

Enable ADC’s Clock

In MCAL projects we should enable peripherels’ clock in advance. Click Mcu -> McuModuleConfiguration -> McuModeSettingConf -> McuModeSettingConf_Run -> McuPartitionConfiguration -> McuPeripheral to enable ADC1’s peripherel clock:

Adc Module’s Configuration

Once we load Adc module, we will find that errors are occurred in AdcInterrupt, one of them is “The number of containers in the AdcInterrupts list is not sufficient.”, while there are only 2 interrupts defined(End of Conversion and WatchDog):

Thus we need to add ADC1 and ADC2’s End of Conversion and WatchDog interrputs, and enable ADC1’s EOC interrupt:

Then we start to configure the Adc module, click Adc -> AdcGeneral for the general configurations, here we need to set the Config Variant to VariantPreCompile and ensure the conversion result of ADC is right aligned:

Click Adc -> AutosarExt to set the Timeout value of Adc module, here we recommend to use system clock, and set the Timeout value to 105984:

Adc Channels Indexes Symbolic Names，Adc Conversion Time Once and Adc Enable CalibrationAPI should be selected:

Click Adc -> AdcConfigSet -> AdcHwUnit, we can change the ADC hardware unit’s name to POT_ADC1_S10 for convenience:

Click POT_ADC1_S10 to configure this ADC hardware unit, the actual input is ADC1’s S10, so we need to choose ADC1 for Adc Hardware Unit, and choose ADC_INTERRUPT for Adc Transfer Type:

Enable Adc Prescaler Value, set the Adc Prescaler Value to 2 and set Adc Calibration Prescale to 4:

Click Conversion resolution, set the resolution to RESOLUTION_10(you can find the explanation of different ADC channel type’s resolution in my previous article - Tips for S32K3’s ADC):

Enable AdcNormalConvTimings:

Click AdcChannel, change the ADC channel’s name to Pot_Adc1_S10 for convenience:

Click Pot_Adc1_S10 to cnfigure the ADC channel, because we are using ADC1’s S10, we should choose S10_ChanNum34 for Adc Physical Channel Name, and input 34 for Adc Physical Channel ID:

Click Pot_Adc1_S10 -> AdcGroup to configuration Adc Group, there are 4 parameters that we need to pay attention:

  1. Adc Group Conversion Mode - in this guide, we trigger ADC conversion in scan mode, so we choose ADC_CONV_MODE_CONTINUOUS;
  2. Adc Group Conversion Type - in this demo, ADC will work in normal mode, so we choose ADC_CONV_TYPE_NORMAL;
  3. Adc Group Trigger Source - we will manually trigger the conversion in software in this demo, so we choose ADC_TRIGG_SRC_SW;
  4. Adc Group Without Interrupts - we will not use any interrupts, so we should enable this item. Click Adc -> AdcConfigSet -> AdcHwUnit -> POT_ADC1_S10 -> AdcGroup -> AdcGroup_0 -> AdcGroupDefinition, choose Pot_Adc1_S10: Here, we finished the configurations of modules that we need to use, save the project and click “Generate code for the currently selected projects” to generate code:
